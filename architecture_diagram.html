Encoder Key Points

  • DCAE-aligned spatial transition: Conv(in, out/4, s=1) + pixel_unshuffle(2). Shortcut: pixel_unshuffle + channel_avg. Both share same spatial rearrangement.
  • Residual scope: shortcut wraps ONLY conv_down + pixel_unshuffle, matching DC-AE. Wavelet concat + out_res_block are OUTSIDE residual.
  • Cascading wavelet: Each block takes LL sub-band (first 3ch) of previous coefficients, applies HaarWT to get next-level coefficients.
  • Per-stage depth: encoder_num_resblocks=[2,3,9,2], so res_block has num-1 ResBlocks before spatial transition, then +1 out_res_block after wavelet concat. Total per stage: 2,3,9,2. Heavy middle at Down2 (9), thin edges at Down0/Down3 (2).
  • I/O shortcut (new): Non-parametric group-average shortcut from mid-block output bypasses norm_out → swish → conv_out. Formula: h.reshape(B, 2*latent_dim, C//(2*latent_dim), H, W).mean(dim=2) then added to conv_out result. HunyuanVAE-aligned. Gated by use_io_shortcut=True.

Decoder Key Points

  • DCAE-aligned spatial transition: Conv(in, out*4, s=1) + pixel_shuffle(2). Shortcut: repeat_interleave + pixel_shuffle. Both share same spatial rearrangement.
  • Residual scope: shortcut wraps ONLY conv_up + pixel_shuffle, matching DC-AE. out_res_block is OUTSIDE residual.
  • Cascading wavelet reconstruction: Each WFUpBlock generates wavelet coefficients, injects previous level LL into current coefficients, then InvHaarWT produces RGB residual w at progressively larger resolutions.
  • Per-stage depth: decoder_num_resblocks=[3,4,12,2], so res_block has num-2 ResBlocks (1, 2, 10, 0) in main trunk after branch split. Total per stage = num counting branch_conv + res_block + out_res_block. Heavy middle at Up2 (12 blocks).
  • I/O shortcut (new): Non-parametric repeat_interleave shortcut from latent z bypasses conv_in. Formula: z.repeat_interleave(base_channels[-1]//latent_dim, dim=1) — for current config, 64→1024 via 16× channel duplication. HunyuanVAE-aligned. Gated by use_io_shortcut=True.
  • No encoder skip connections: Decoder is fully independent, all information comes from latent z.

WFDownBlock Design

  • DCAE residual alignment: Both paths use pixel_unshuffle for spatial rearrangement. Conv operates at ORIGINAL resolution before pixel_unshuffle, seeing full spatial context.
  • Minimal residual scope: Shortcut wraps ONLY conv_down + pixel_unshuffle. Wavelet concat and out_res_block process shortcut-enhanced features OUTSIDE residual.
  • Wavelet energy flow: Cascading HaarWT on LL sub-band produces multi-level frequency decomposition. Flow is concatenated not added to main trunk, then fused by out_res_block.

WFUpBlock Design

  • DCAE residual alignment: Both paths use pixel_shuffle for spatial rearrangement. Addition happens IMMEDIATELY after spatial expansion.
  • Branch-then-split: branch_conv expands channels to in_ch + efs, then splits. Model learns how to partition information between spatial path and wavelet flow.
  • Cascading wavelet reconstruction: Each block generates 12-channel wavelet coefficients, injects previous level LL residual w into first 3 channels, then InvHaarWT produces next RGB residual at 2x resolution.
  • out_res_block identity shortcut: After DCAE addition, in_ch == out_ch for this ResBlock since pixel_shuffle already mapped to out_ch. No channel projection needed.

Wavelet Flow Mechanism

  • Encoder: Builds a multi-level Haar wavelet decomposition tree. At each level, takes the LL sub-band first 3 channels = low-frequency RGB approximation and applies another HaarWT. The resulting 12-channel coefficients are (1) processed by in_flow_conv and concatenated to the spatial trunk, and (2) passed to the next level.
  • Decoder: Reconstructs the wavelet tree in reverse. Each WFUpBlock generates 12-channel coefficients from latent features, injects previous level LL residual w, then InvHaarWT produces 3-channel RGB image at 2x resolution. This w cascades upward until the final output merge.
  • Wavelet loss: L1 loss between encoder and decoder coefficients reversed order, weight=0.1. Forces decoder to reconstruct frequency-domain structure consistent with encoder decomposition.
  • No skip connections: Decoder wavelet flow is entirely self-generated from latent. No encoder coefficients are passed to decoder at inference.

Pipeline notes

  • Two disc forwards per gen step: in multiscale mode, the generator step runs D on both reconstruction and real (the latter detached) because feature matching needs paired intermediate features. This adds ~1 extra disc forward compared to single-scale.
  • Adaptive d_weight is untouched: g_loss is aggregated to a scalar before gradient computation, so calculate_adaptive_weight still sees a single scalar and its logic is identical to single-scale.
  • Feature matching is outside the adaptive weight: fm_loss uses a fixed λ_feat=10.0 multiplier, not the adaptive d_weight. This mirrors pix2pixHD.
  • Alternating gen/disc: discriminator step is a separate call with optimizer_idx=1, runs D(real.detach()) and D(rec.detach()) and computes per-scale hinge loss averaged across scales.

Multi-scale design

  • Independent parameters: each scale has its own NLayerDiscriminatorWithFeat with independent weights. No weight sharing across scales.
  • Progressive downsampling: AvgPool2d(3, stride=2, padding=[1,1], count_include_pad=False) between scales. For 1024px input, the three scales see 1024→512→256.
  • Why multi-scale: a single PatchGAN at full resolution has a limited receptive field (~70×70 for n_layers=3), missing coarse structure at high resolution. Downsampled scales give the disc an explicit "zoom-out" view.
  • Output convention: result[0] is full-res, result[num_D-1] is the most downsampled (pix2pixHD uses num_D-1-i indexing internally).

PatchGAN details

  • Normalization: BatchNorm2d on all layers except model0 (first Conv) and model4 (final logit). Aligns with pix2pixHD original.
  • Penultimate stride change: model3 uses stride=1 (vs stride=2 elsewhere) so the final receptive field stays moderate (~70×70 effective for n_layers=3).
  • Feature extraction: because each block is registered as self.modelN separately, forward can collect outputs progressively into a list. Without this split, FM loss cannot access intermediate activations.
  • Small-batch concern: BatchNorm statistics are noisy at BS=2 in DDP (no SyncBN). If 1024px training is unstable, consider porting InstanceNorm2d — left as future work.

7.4 Loss formulas

# Total generator loss
total_loss = nll_loss / exp(logvar) + logvar
           + kl_weight · kl_loss
           + d_weight · disc_factor · g_loss
           + wavelet_weight · wavelet_loss
           + fm_loss                                     // ← new in multiscale mode

# Multi-scale g_loss (average over scales)
pred_fake = D(reconstructions)            # List[List[Tensor]]
g_loss = (1/num_D) · Σ_i [ -mean(pred_fake[i][-1]) ]

# Feature matching loss (L1 on intermediate features)
pred_real = D(inputs.detach())            # extra D forward on real
feat_weights = 4 / (n_layers_D + 1)      # per-layer weight
D_weights    = 1 / num_D                   # per-scale weight

fm_loss = 0
for i in range(num_D):                 # all scales
    for j in range(n_layers_D + 1):      # all layers except final logit
        fm_loss += D_weights · feat_weights · L1(
             pred_fake[i][j],                           # gradient flows to generator
             pred_real[i][j].detach()      # real features = frozen target
        )
fm_loss *= feat_match_weight            # λ_feat = 10.0 (pix2pixHD default)

# Discriminator step — multi-scale hinge loss
pred_real = D(inputs.detach())
pred_fake = D(reconstructions.detach())
d_loss = (1/num_D) · Σ_i [ hinge_d_loss(
                      pred_real[i][-1], pred_fake[i][-1]
                     ) ]

# hinge_d_loss = 0.5 · ( relu(1 - real) + relu(1 + fake) )

Default values (aggressive route, currently deployed)

  • num_D = 3 (scales: full-res, /2, /4)
  • n_layers_D = 3 (conv depth per PatchGAN)
  • ndf = 64 (base disc channel count)
  • feat_match_weight (λ_feat) = 10.0
  • disc_weight = 0.5 (unchanged)
  • disc_loss = "hinge" (unchanged)
  • disc_start = 5 (unchanged — disc activates from step 5 onward)
  • Disc params: single-scale 2.77 M → multi-scale 8.30 M (~3×)

Coexistence — not a replacement

  • The original NLayerDiscriminator in discriminator.py is untouched. Run DISC_TYPE=single bash train_wfivae.sh to use it and get identical behavior to pre-patch runs.
  • train_wfivae.sh auto-suffixes EXP_NAME with _single/_multiscale, so checkpoints and CSV from the two variants live in disjoint output directories and can be trained side-by-side.
  • Checkpoints are not cross-compatible (param shapes differ). Resuming with the wrong disc_type raises a PyTorch shape-mismatch at load time.
  • Ablation knobs: FEAT_MATCH_WEIGHT=0 disables FM loss while keeping the multi-scale GAN loss; NUM_D=2 drops to 2 scales to save memory.
