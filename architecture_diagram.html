Energy flow switch

  • What is kept: W^(1) Haar WT at encoder entry (3→12ch, ÷2) and Inv Haar WT at decoder exit (12→3ch, ×2). These drive the input-side 2× compression and are part of the backbone, not the mid-layer pathway.
  • What is removed (vs paper WF-VAE): per-stage in_flow_conv / out_flow_conv, the +energy_flow_size channel expansion in branch_conv, the cascading h[:,:3] += w wavelet residual injection, enc_coeffs/dec_coeffs lists, and the WL loss.
  • Code switch: use_energy_flow=false in examples/wfivae2-image-32x-192bc.json. Flipping to true reproduces the paper form on the same backbone — the code path is preserved. No separate no-ef config ships.
  • Parameter savings: ~22 M (−4.5%) vs paper DCAE + ef (489.88 M → 467.80 M). Decoder drops ~15 M (branch_conv channel expansion alone); encoder drops ~7 M (four in_flow_conv removals).

Encoder Key Points

  • W^(1) HaarWT is retained: the input-side Haar wavelet transform is part of the backbone (drives 2× compression), not the removed mid-layer energy flow. It fires once per encoder call, turning 3ch RGB into 12ch at half spatial resolution.
  • No mid-layer energy flow: each WFDownBlock no longer runs wavelet_transform(w[:,:3]), in_flow_conv, or a concat. The trunk goes res_block → (conv_down + pixel_unshuffle + DCAE shortcut) → out_res_block directly, and out_res_block.in_channels = out_ch (not out_ch + ef). Encoder.forward returns (h, None).
  • DCAE-aligned spatial transition: Conv(in, out/4, s=1) + pixel_unshuffle(2). Shortcut: pixel_unshuffle + channel_avg. Both share the same spatial rearrangement.
  • Residual scope: shortcut wraps ONLY conv_down + pixel_unshuffle. out_res_block processes shortcut-enhanced features OUTSIDE the residual.
  • Per-stage depth: encoder_num_resblocks=[4,4,4,2] — 4 ResBlocks at Down0/1/2, only 2 at Down3 (pre-spatial res_block has num-1, plus 1 out_res_block).
  • ECA channel attention: every ResnetBlock2D ends with ECALayer(C) before the residual add (Wang et al. CVPR 2020). Adaptive kernel k = |log2(C)/2 + 0.5|_odd. At C ∈ {256, 512, 1024} k=5, ~5 params per block.
  • I/O shortcut: non-parametric group-average shortcut from mid-block output bypasses norm_out → swish → conv_out. Formula: h.reshape(B, 2*latent_dim, C//(2*latent_dim), H, W).mean(dim=2). HunyuanVAE-aligned.

Decoder Key Points

  • W^(1) InvHaarWT is retained: output-side inverse Haar wavelet transforms the 12ch feature map back into 3ch RGB at 2× resolution. Part of the backbone; not tied to energy-flow toggling.
  • No mid-layer energy flow: branch_conv keeps channels at in_ch (no +128 expansion). No split, no out_flow_conv, no per-stage InverseHaarWaveletTransform2D, no coeffs[:,:3] += w injection, no final h[:,:3] += w before the output IWT. Decoder.forward returns (dec, None).
  • DCAE-aligned spatial transition: Conv(in, out*4, s=1) + pixel_shuffle(2). Shortcut: repeat_interleave + pixel_shuffle. Addition happens immediately after spatial expansion.
  • Per-stage depth: decoder_num_resblocks=[5,5,5,3] — 5 ResBlocks at Up0/1/2, only 3 at Up3. Main-trunk res_block has num-2 ResBlocks (3,3,3,1), plus 1 branch_conv + 1 out_res_block. Up3 was shallowed because its middle ResBlocks run at 512ch × 256² — the single largest activation footprint in the whole decoder.
  • ECA channel attention: every decoder ResnetBlock2D (including branch_conv, main-trunk res_block, out_res_block, and mid ResnetBlock2D) ends with ECALayer(C) — ~5 params each.
  • I/O shortcut: non-parametric repeat_interleave shortcut from latent z bypasses conv_in. Formula: z.repeat_interleave(base_channels[-1]//latent_dim, dim=1) — for current config, 64→1024 via 16× channel duplication. HunyuanVAE-aligned.
  • No encoder skip connections: decoder is fully independent, all information comes from latent z.

WFDownBlock Design

  • DCAE residual alignment: both paths use pixel_unshuffle for spatial rearrangement. Conv operates at ORIGINAL resolution before pixel_unshuffle, seeing full spatial context.
  • Minimal residual scope: shortcut wraps ONLY conv_down + pixel_unshuffle. out_res_block processes shortcut-enhanced features OUTSIDE the residual.
  • No mid-layer energy flow: there is no HaarWaveletTransform2D call inside this block, no in_flow_conv, no concat. The trunk becomes a clean res_block → DCAE spatial transition → out_res_block pipeline. out_res_block.in_channels = out_ch (not out_ch + ef).
  • forward signature: kept as forward(x, w=None) for interface symmetry with the ef=true code path. Returns (h, None) under use_energy_flow=false.

WFUpBlock Design

  • DCAE residual alignment: both paths use pixel_shuffle for spatial rearrangement. Addition happens IMMEDIATELY after spatial expansion.
  • No mid-layer energy flow: branch_conv keeps channels at in_ch (no +128 expansion), there is no split, no out_flow_conv, and no InverseHaarWaveletTransform2D. The trunk becomes branch_conv → res_block → DCAE spatial transition → out_res_block.

6. Discriminator & Adversarial Loss (pix2pixHD) Training

Two discriminator variants coexist — switch via --disc_type {single,multiscale}. The legacy single-scale PatchGAN lives in discriminator.py; the pix2pixHD-style multi-scale disc + feature-matching loss live in multiscale_discriminator.py. Both variants support four normalization modes (--disc_norm {sn,bn,in,none}); on this branch the default is sn (Miyato et al., Spectral Normalization, ICLR 2018). This is batch-size independent, crucial at BS=1/1024px.

Pipeline notes

  • WL loss is inert under no-ef: WFIVAE2Model.forward returns extra_output=None (not (None, None) — which would be truthy); the train loop keeps wavelet_coeffs = None; LPIPSWithDiscriminator.forward takes the else branch and sets wl_loss = torch.tensor(0.0). The CSV column wavelet_loss stays at 0 across the entire run. No code change needed in the loss class or the training loop.
  • Two disc forwards per gen step: in multiscale mode, the generator step runs D on both reconstruction and real (the latter detached) because feature matching needs paired intermediate features. This adds ~1 extra disc forward compared to single-scale.
  • Adaptive d_weight is untouched: g_loss is aggregated to a scalar before gradient computation, so calculate_adaptive_weight still sees a single scalar and its logic is identical to single-scale.
  • Feature matching is outside the adaptive weight: fm_loss uses a fixed λ_feat=10.0 multiplier, not the adaptive d_weight. This mirrors pix2pixHD.
  • Alternating gen/disc: discriminator step is a separate call with optimizer_idx=1, runs D(real.detach()) and D(rec.detach()) and computes per-scale hinge loss averaged across scales.

Multi-scale design

  • Independent parameters: each scale has its own NLayerDiscriminatorWithFeat with independent weights. No weight sharing across scales.
  • Progressive downsampling: AvgPool2d(3, stride=2, padding=[1,1], count_include_pad=False) between scales. For 1024px input, the three scales see 1024→512→256.
  • Why multi-scale: a single PatchGAN at full resolution has a limited receptive field (~70×70 for n_layers=3), missing coarse structure at high resolution. Downsampled scales give the disc an explicit "zoom-out" view.
  • Output convention: result[0] is full-res, result[num_D-1] is the most downsampled (pix2pixHD uses num_D-1-i indexing internally).

PatchGAN details

  • Normalization (default sn): every Conv2d is wrapped with spectral_norm; the norm slot between Conv and LReLU is nn.Identity(). State_dict gains weight_orig/weight_u/weight_v keys. Switch via DISC_NORM={sn,bn,in,none}; bn restores the legacy BatchNorm2d behavior (noisy at BS=1/2 in DDP without SyncBN).
  • Penultimate stride change: model3 uses stride=1 (vs stride=2 elsewhere) so the final receptive field stays moderate (~70×70 effective for n_layers=3).
  • Feature extraction: because each block is registered as self.modelN separately, forward can collect outputs progressively into a list. Without this split, FM loss cannot access intermediate activations.
  • SN Lipschitz constraint: spectral_norm normalizes each Conv's weight by its top singular value (power iteration, Ip=1), enforcing a per-layer Lipschitz bound of 1. This yields training stability comparable to WGAN-GP without the gradient-penalty cost.

6.4 Loss formulas

# Total generator loss
total_loss = nll_loss / exp(logvar) + logvar
           + kl_weight · kl_loss
           + d_weight · disc_factor · g_loss
           + wavelet_weight · wavelet_loss       // = 0 under use_energy_flow=false
           + fm_loss                                     // pix2pixHD feature matching

# Multi-scale g_loss (average over scales)
pred_fake = D(reconstructions)            # List[List[Tensor]]
g_loss = (1/num_D) · Σ_i [ -mean(pred_fake[i][-1]) ]

# Feature matching loss (L1 on intermediate features)
pred_real = D(inputs.detach())            # extra D forward on real
feat_weights = 4 / (n_layers_D + 1)      # per-layer weight
D_weights    = 1 / num_D                   # per-scale weight

fm_loss = 0
for i in range(num_D):                 # all scales
    for j in range(n_layers_D + 1):      # all layers except final logit
        fm_loss += D_weights · feat_weights · L1(
             pred_fake[i][j],                           # gradient flows to generator
             pred_real[i][j].detach()      # real features = frozen target
        )
fm_loss *= feat_match_weight            # λ_feat = 10.0 (pix2pixHD default)

# Discriminator step — multi-scale hinge loss
pred_real = D(inputs.detach())
pred_fake = D(reconstructions.detach())
d_loss = (1/num_D) · Σ_i [ hinge_d_loss(
                      pred_real[i][-1], pred_fake[i][-1]
                     ) ]

# hinge_d_loss = 0.5 · ( relu(1 - real) + relu(1 + fake) )

Default values (currently deployed)

  • disc_type = multiscale (pix2pixHD)
  • disc_norm = sn (Spectral Normalization — batch-size independent)
  • num_D = 3 (scales: full-res, /2, /4)
  • n_layers_D = 3 (conv depth per PatchGAN)
  • ndf = 64 (base disc channel count)
  • feat_match_weight (λ_feat) = 10.0
  • disc_weight = 0.5
  • disc_loss = "hinge"
  • disc_start = 5 (disc activates from step 5 onward)
  • Disc params: single-scale 2.77 M → multi-scale 8.30 M (~3×)
