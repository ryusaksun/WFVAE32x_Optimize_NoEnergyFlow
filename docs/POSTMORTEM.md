## 如何使用本文档

### 开发前检查 (Pre-Development)
1. 识别即将修改的文件
2. 在本文档中搜索这些文件
3. 阅读相关条目的"防护规则"
4. 确保新代码不违反这些规则

### 修复 Bug 后 (Post-Fix)
1. 分析本次修复的根因
2. 按模板格式添加新条目
3. 更新统计概览和分类索引
4. Commit 信息中添加 `Postmortem: PM-XXX`

### 推荐的 Commit 消息格式
```
fix(scope): 简短描述

详细说明问题和修复方案

Postmortem: PM-XXX
```
