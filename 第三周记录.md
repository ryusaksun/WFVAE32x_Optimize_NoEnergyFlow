## 2.判别器启动过早的问题

由于论文中使用了对抗损失，会根据重建的梯度和对抗的梯度来动态调整对抗损失的权重，所以论文中在开始时就启动了判别器。

如果判别器启动过早会导致多个严重的训练问题。让我从代码和理论两个角度分析：

### 早期启动判别器的问题

#### 1. 模式崩溃（Mode Collapse）

**现象：**
- 生成器输出的图像变得模糊、单一化
- 重建图像失去细节，趋向于"安全"的平均模式
- PSNR 可能还可以，但感知质量很差

**原因（`perceptual_loss_2d.py:136-152`）：**

```python
# 生成器损失
g_loss = -torch.mean(logits_fake)  # 希望判别器认为重建是真的
loss = weighted_nll_loss + kl_weight * kl_loss + d_weight * disc_factor * g_loss
```

当生成器还没学会基本重建时：
- `weighted_nll_loss` 很大（重建很差）
- 判别器很容易区分真假
- 生成器会优先满足判别器而非重建质量
- 导致输出"看起来真实"但与输入不匹配

#### 2. 梯度不稳定/消失

**现象：**
- 训练损失剧烈震荡
- 学习率需要大幅降低
- 可能出现 NaN 或 Inf

**原因（`perceptual_loss_2d.py:79-88`）：**

```python
def calculate_adaptive_weight(nll_loss, g_loss, last_layer):
    nll_grads = torch.autograd.grad(nll_loss, layer)[0]
    g_grads = torch.autograd.grad(g_loss, layer)[0]
    d_weight = norm(nll_grads) / (norm(g_grads) + 1e-6)  # 自适应权重
```

早期阶段：
- `nll_grads` 可能很大（重建差）
- `g_grads` 也很大（判别器强）
- 比值不稳定，导致 `d_weight` 剧烈波动

#### 3. 判别器过强（Discriminator Overpowering）

**现象：**
- 判别器损失接近 0
- `logits_real` 接近很大正数，`logits_fake` 接近很大负数
- 生成器梯度消失，无法学习

**原因（`perceptual_loss_2d.py:169-176`）：**

```python
# 判别器很容易区分早期的垃圾重建
logits_real = discriminator(inputs)      # → 接近 +∞
logits_fake = discriminator(recon)       # → 接近 -∞
d_loss = hinge_loss(logits_real, logits_fake)  # → 0
```

Hinge loss 问题：

```python
loss_real = relu(1 - logits_real)  # logits_real >> 1 → 0
loss_fake = relu(1 + logits_fake)  # logits_fake << -1 → 0
```

判别器"太聪明"后停止学习，生成器得不到有用的反馈。

#### 4. KL 散度崩溃

**现象：**
- `kl_loss` 降到接近 0
- 潜在空间失去变分性质，退化为确定性编码
- 后验分布方差 → 0

**原因：**

```python
kl_loss = posteriors.kl()  # KL(q(z|x) || N(0,1))
loss = ... + 1e-6 * kl_loss + ... + d_weight * g_loss
```

当对抗损失主导时：
- 生成器忽略 KL 正则化（权重太小 1e-6）
- 为了最大化判别器欺骗能力，潜在空间过拟合
- VAE 退化为普通自编码器

#### 5. 小波能量流失衡

**现象（针对此项目特有）：**

- 编码器和解码器的小波系数差异巨大
- 高频细节丢失或过度增强
- 重建图像出现伪影

**原因（`perceptual_loss_2d.py:121-133`）：**

```python
if wavelet_coeffs:
    wl_loss = L1(enc_coeffs, dec_coeffs_reversed)
    loss = ... + 0.1 * wl_loss
```

早期启动判别器时：
- 判别器可能利用小波域的高频伪影作为"假图"的标志
- 生成器被迫抑制高频，导致模糊
- 能量流路径被破坏

### 实验证据推理

假设 `disc_start=0`（立即启动）vs `disc_start=5000`：

| 指标 | disc_start=0 | disc_start=5000 |
|------|-------------|-----------------|
| Step 1000 PSNR | 15-18 dB（很差） | 22-25 dB（合理） |
| Step 1000 LPIPS | 0.4-0.6（模糊） | 0.2-0.3（清晰） |
| 训练稳定性 | 震荡，可能崩溃 | 平滑上升 |
| 重建保真度 | 低（只追求"真实感"） | 高（先学会复制） |
| 潜在空间质量 | 混乱 | 结构化 |

### 从代码验证

查看判别器损失的判断（`train_image_ddp.py:420-431`）：

```python
if current_step % 2 == 1 and current_step >= disc.module.discriminator_iter_start:
    # 只有满足步数要求才训练判别器
```

这个设计确保：
1. 前 5000 步：生成器专注学习 inputs → latent → recon 的基本映射
2. 5000 步后：重建已经合理，判别器才介入细化细节

### 最佳实践

判别器启动时机选择：

| 分辨率 | 推荐 disc_start | 理由 |
|--------|-----------------|------|
| 256px | 2000-3000 | 重建相对简单 |
| 512px | 3000-5000 | 需要更多预热 |
| 1024px | 5000-10000 | 高分辨率需要充分预训练 |

判断指标（启动判别器前应满足）：
- PSNR > 20 dB
- LPIPS < 0.3
- 重建图像肉眼可辨认内容

### 总结

过早启动判别器就像让一个刚学会走路的婴儿去跑马拉松：
- **生成器**：还没学会"复制"输入，就被要求"创造"真实图像
- **判别器**：面对垃圾输出，要么过强（梯度消失），要么被误导
- **结果**：训练崩溃或收敛到糟糕的局部最优

`disc_start=5000` 的设计是经验和理论的结合，确保对抗训练在一个稳定的基础上进行。
